### 4.이벤트 조회 및 수정 REST API 개발
#### 이벤트 수정 API 구현

##### 이벤트 수정시 구현해야할 테스트
1. 이벤트를 정상적으로 수정하기
2. 입력값이 비어있는 경우에 이벤트 수정 실패(JSR303) 
3. 입력값이 잘못된 경우에 이벤트 수정 실패 (비즈니스 로직상 입력값이 잘못된경우)
4. 존재하지 않는 이벤트 수정 실패(수정할 id 값이 존재 하지 않는경우)
```
내가 처음에 강의를 안듣고 구현을 했을때는 1,4번 케이스만 테스트에서 구현했고 
Controller 단에서는 2,3 번도 구현을 해줬지만 테스트는 작성하지 않았다.
케이스 마다 전부 테스트 케이스 작성하는 버릇을 들어야 겠다.
```
